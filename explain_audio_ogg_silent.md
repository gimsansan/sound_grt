# 🐛 이슈 기록: 안드로이드 기기 오디오 무음 현상 및 해결

## 🚨 문제 현상 (Issue)
* **증상**: 앱 내에서 오디오 재생을 실행하면 터미널 로그상으로는 에러가 없고 정상적으로 재생이 시작된 것처럼 보이나, 실제 **안드로이드 디바이스에서는 아무런 소리가 나지 않음**.
* **로그 분석**:
  ```text
  LOG 🎵 AudioManager: ukulele_ab3 재생됨
  LOG ⏰ AudioManager: ukulele_ab3 백업 정리됨
  ```
  정상적인 상황에서는 소리 재생이 끝난 후 `자동 정리됨`이 출력되어야 하지만, `백업 정리됨`이 출력됨. 이는 Android의 오디오 플레이어(MediaPlayer)가 파일을 끝까지 재생하지 못해 1.5초 타이머가 강제로 작동했음을 의미함.

## 🔍 원인 파악 (Root Cause)
1. **음원 파일 손상 및 가짜 파일 (주 원인)**: 
   * `assets/sounds/`에 존재하던 기존 `.ogg` 파일들의 용량을 확인한 결과 **단 14바이트**에 불과했음.
   * 파일을 텍스트 에디터로 열어본 결과 오디오 데이터가 아닌 HTML 오류 텍스트인 `404: Not Found`만 적혀 있었음. 즉, 이전 다운로드 스크립트의 링크가 만료되어 껍데기만 다운로드된 상태였음.
2. **Android의 OGG 코덱 호환성 한계**:
   * 설령 정상적인 Ogg 파일이었더라도, 안드로이드의 `expo-av`는 특정 인코딩(예: Opus)이나 비표준 샘플 레이트를 가진 `.ogg` 파일에 대해 에러를 던지지 않고 조용히 재생을 실패하는(Silent Failure) 고질적인 문제가 있음.

## 🛠 해결 방안 (Resolution)
1. **정상적인 음원 소스 확보**: 
   * GitHub의 검증된 사운드 폰트 저장소(`gleitz/midi-js-soundfonts`)에서 호환성이 높은 `.mp3` 원본 파일들을 새로 다운로드함.
2. **모바일 최적화 포맷으로 일괄 변환 (FFmpeg 사용)**:
   * 안드로이드와 완벽하게 호환되며 용량 대비 음질이 뛰어난 **`.m4a` (AAC 코덱)** 형식으로 파일들을 일괄 변환함.
   * 빠른 재생 시작을 위해 `+faststart` (moov atom 최적화) 옵션을 적용하여 딜레이를 최소화함.
3. **코드 업데이트**: 
   * `constants/instruments.ts` 내의 모든 `require` 경로 확장자를 `.ogg`에서 `.m4a`로 변경 완료.

## 📌 교훈 및 향후 지침
* 오디오 재생이 소리 없이 실패할 경우, 코드의 문제보다 **오디오 파일 자체의 용량과 포맷 무결성**을 가장 먼저 의심하고 점검해야 함.
* 모바일(특히 안드로이드) 앱 개발 시 효과음 포맷은 가급적 가장 안전하고 보편적인 **`.m4a` (AAC)** 또는 **`.mp3`**를 사용하는 것을 표준으로 삼음.
