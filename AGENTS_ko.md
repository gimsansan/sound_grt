# AI 에이전트 규칙 및 컨텍스트

## 대상 플랫폼: 오직 Android

- **Android 전용**: 이 프로젝트는 오직 Android 모바일 환경을 위해서만 설계되고 개발되었습니다.
- **iOS 컨텍스트 배제**: iOS 전용 코드, 설정, 예외 처리(예: `playsInSilentModeIOS: true`)를 제안하거나 설명하거나 작성하지 마세요. 전적으로 Android 기능 및 문제 해결에만 집중하세요.

## 코드 분석 및 데드 코드(Dead Code) 규칙

- **데드 코드 식별**: 단순히 어딘가에 `import` 되어 있다는 이유만으로 해당 파일이 "사용 중"이라고 맹목적으로 가정하지 마세요. 다음 두 가지를 반드시 확인해야 합니다: (1) 메인 앱 라우터(`app/` 디렉터리)에서 접근 가능한가? (2) export된 값이나 훅(hook)이 컴포넌트 내에서 실제로 소비/사용되고 있는가? (예: 앱을 감싸는 Context `Provider`라 하더라도 그 값이 읽히지 않는다면 데드 코드입니다).
- **고립된 코드(Island Code) 주의**: 특정 기능 폴더 내에서 순환적으로 서로만 `import` 하는 파일 군집을 주의하세요. 이 군집이 메인 라우터와 완전히 단절되어 있다면 데드 코드로 분류하세요.

## 현재 프로젝트 컨텍스트

- **활성화된 메인 화면**: 현재 유일하게 활성화된 화면은 `app/(tabs)/sound_grt/index.tsx` (피아노 건반 탭)입니다. 단어 게임, 미니 게임 등 이 화면과 직접적/간접적으로 연결되지 않은 다른 탭의 나머지 요소들은 현재 시점에서 모두 '데드 코드'로 간주됩니다.

## 자동 학습 및 자체 수정

- **선제적 문서화**: 이 `AGENTS.md` 파일은 당신의 영구적인 메모리이자 시스템 프롬프트의 확장으로 작동합니다.
- **오류/학습 시 즉각적인 조치**: 새로운 문제(예: 버그, 라이브러리 충돌, 놓친 컨텍스트, 데드 코드 분석 실패 등)에 직면하거나 사용자가 당신의 실수를 정정해 줄 때마다, 사용자가 지시할 때까지 기다리지 말고 즉각적이고 선제적으로 파일 편집 도구를 사용하여 `AGENTS.md`에 근본 원인과 해결책/우회 방법을 추가해야 합니다.
- **언어 정책**: LLM 시스템의 최대한의 논리적 명확성과 엄격한 준수를 보장하기 위해 원본 `AGENTS.md`의 모든 항목은 명확하고 간결한 영어로 작성되어야 합니다.
- **선제적 실행 및 상태 변경 금지**: 사용자가 질문을 할 때, 오직 답변만 제공하세요. 사용자가 명시적으로 해당 조치를 요청하지 않는 한 임의로 터미널 명령을 실행, 제안하거나 상태 변경(예: 브랜치 전환)을 유발하지 마세요. 상태를 변경하는 행동은 단순한 이론적 질문이 아니라 사용자의 명확하고 직접적인 명령이 필요합니다.
- **엄격한 구조 준수**: 사용자가 제공한 자료(예: 이미지, 다이어그램, 목록 등)의 요소를 맵핑하거나 설명할 때, 제시된 정확한 번호 매기기, 구분, 구조를 엄격하게 유지하세요. 명시적인 지시가 없는 한 개별 항목을 임의로 병합, 누락, 그룹화(예: 3번과 4번을 하나로 합치는 등)하지 마세요. 엄격한 1:1 맵핑을 보장하세요.
- **기술적 정확성 및 정밀성**: 기술적 사실을 과장하거나 왜곡하지 마세요; 무엇보다 정확성을 최우선으로 하세요. 코드의 '논리적 제어 흐름(소프트웨어적 대기/지연)'과 시스템 레벨의 '물리적 자원 점유(스레드 차단/UI 프리징)'를 엄격하게 구분하세요.
